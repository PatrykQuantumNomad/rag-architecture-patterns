- **TOOL-01** (was: 03-03 v1.1 item): Add `addopts = "-m 'not live'"` to `pyproject.toml [tool.pytest.ini_options]` so a bare `pytest path/to/file.py -v` (no `-m` flag) cannot accidentally consume API budget. Update live-test invocation pattern in plans + CLAUDE.md to use `-m live -k <test>` explicitly.
- **TOOL-02** (was: 02-04 / 03-03 / 05-02 v1.1 item): Augment `score.py`'s `token_usage_parser=get_token_usage_for_openai` to parse usage from LiteLLM `ModelResponse` bodies even when the parser misses some calls — currently RAGAS judge cost ledgers report $0 despite real spend. (Phase 8 mitigates by writing fallback `estimator: {...}` block; root fix still wanted.)
- **TOOL-03** (was: 05-02 v1.1 item): `tracker.persist()` should honor caller-supplied `--results-dir` / `--output-dir` — `evaluation/harness/run.py:273` and `evaluation/harness/score.py:518` call `tracker.persist()` with no `dest_dir` arg, hardcoding the destination. Manifested as monkeypatch fix in Plan 05-02 live test. v1.1 plumb-through: pass `dest_dir=Path(args.output_dir) / 'costs'` in both call sites + add `--cost-dir` flag at top-level `pipeline.py`.
- **TOOL-04** (was: 07-01 v1.1 item): `tier-4-multimodal/scripts/eval_capture.py` should prepend `Path(sys.executable).parent` to `os.environ['PATH']` BEFORE instantiating `RAGAnything`, so `RAGAnything._ensure_lightrag_initialized()`'s mineru parser-installation probe finds `.venv/bin/mineru` regardless of parent shell. Currently fails silently to degraded captures. ~2 LOC patch at top of `eval_capture.py main()`.
- **TOOL-05** (NEW from v1.0): RAGAS 0.4.3 `from ragas.metrics import context_precision` `DeprecationWarning` — track migration to new RAGAS API path.
- **TOOL-06** (NEW from v1.0): `compare.py` emits `**Embedder by tier:**` (markdown bold) instead of `## Embedder by tier` (heading). Plan-spec verifier regex was wrong; compare.py was RAW-LOCKED so no fix applied during v1.0. Cosmetic only.
- **TOOL-07** (NEW from v1.0): Cost-window selection in verifier scripts — prefer ts-window grep over `ls -t | head -1` to avoid pytest test-bleed pollution (5 zero-byte placeholder cost JSONs from post-sweep test runs live in `evaluation/results/costs/`).

## Out of Scope (v1.0 — preserved for v1.1 reasoning)

| Feature | Reason |
|---------|--------|
| Adding new tiers / new RAG architectures | v1.0 was fix-and-ship the existing 5; the blog narrative is built around them |
| Adding new questions to the golden set | 30 is locked (single-hop / multi-hop / multimodal split is the blog's evidence structure) |
| Statistical-significance testing | Sample size (30 × 5) is too small; magnitude-only is the blog's framing (METH-01 partial mitigation in v1.1) |
| Consolidating `tier-N-name/` ↔ `tier_N_name/` folders | Underscore folders are intentional setuptools shims; CLEANUP-01 in v1.1 |
| Full self-grading bias mitigation (re-running everything with multiple judges) | Cost-prohibitive; spot-check **measurement** in v1.0, full mitigation METH-02 in v1.1+ |
| Re-architecting evaluation harness, cost-tracker, or shared layer | v1.0 minimized new abstractions; only added `freeze.py` + `pipeline.py` + `multi_judge_spotcheck.py` + `smoke_gate.py` + `diagnostics.py` |
| Switching providers (e.g. moving Tier 1 off OpenRouter) | Existing provider mix locked for v1.0 |
| Cross-language ports of any tier | Python-only; locked by `pyproject.toml` |
