<objective>
Build the autonomous Tier 4 graphml rebuild stack: a wipe-and-rebuild script (`ingest_from_mineru.py`) that calls RAG-Anything's public `insert_content_list` API to reload the 75 papers already parsed in `tier-4-multimodal/output/` (Stage 1 of RESEARCH.md data flow), and a provenance writer (`log_graph_stats.py`) that captures node/edge counts, kv_store cardinalities, and pinned library versions to `evaluation/results/diagnostics/tier-4-graph-stats-{TS}.json` (Stage 2). Includes unit tests covering the `_absolutize_image_paths` helper (Pitfall 2 of RESEARCH.md), the `_content_list_v2.json` exclusion (Anti-pattern of RESEARCH.md), the `--reset` wholesale-rmtree contract (Pitfall 4 of RESEARCH.md), and the provenance JSON Pydantic shape.

Purpose: This plan satisfies success criteria #1, #2, and #5 of the phase. It produces the on-disk graph that Plan 02-03's smoke test will query against, and the provenance manifest that Phase 7 + Phase 9 will cite verbatim. It is autonomous because RAG-Anything's `insert_content_list` makes only HTTP calls (entity-extraction LLM + embeddings + vision), all of which work inside the sandbox — only the upstream MineRU CLI invocation requires the host (handled separately by Plan 02-02).

Output: A populated `rag_anything_storage/tier-4-multimodal/` directory with `graph_chunk_entity_relation.graphml` + 8 `kv_store_*.json` + 3 `vdb_*.json` files, plus a `tier-4-graph-stats-{TS}.json` provenance manifest.
