# Milestone v1.0: Eval Handoff

**Status:** ✅ SHIPPED 2026-05-08
**Phases:** 1–9
**Total Plans:** 21
**Tag:** `v1.0`

## Overview

Fix two ship-blocker bugs (Tier 5 `empty_contexts`, Tier 4 graphml regen), add a small amount of harness scaffolding (`freeze.py` + `pipeline.py` + per-row NaN reason + embedder provenance), then re-run all 5 tiers on a single date with a single git SHA, run a multi-judge spot-check to bound family bias, and produce one immutable frozen markdown doc (`evaluation/results/frozen/eval-numbers-v1.0.md`) that copy-pastes into the external blog repo with full provenance and honest disclaimers.

Phases 1 and 2 were parallel-friendly bug fixes. Phases 3–6 were parallel-friendly harness/provenance work. Phases 7–9 were strictly sequential and gated on everything before them.

**Issues Deferred to v1.1:**

- LiteLLM judge token-usage parser misses OpenRouter responses → all `ragas-judge-*.json` cost ledgers report $0 (real spend ~$0.45/sweep). Mitigated in P8 via `_estimate_cost_fallback`. Workaround acceptable; root fix is v1.1 hardening.
- `tracker.persist()` ignores caller's `--results-dir` (run.py:273, score.py:518); cost files always land in hardcoded `DEFAULT_COSTS_DIR`. Tests work around via monkeypatch.
- `pyproject.toml` `addopts = "-m 'not live'"` not set → bare pytest invocations can accidentally consume API budget.
- `tier-4-multimodal/scripts/eval_capture.py` should prepend `Path(sys.executable).parent` to `os.environ['PATH']` so `RAGAnything._ensure_lightrag_initialized()`'s mineru parser-installation probe finds `.venv/bin/mineru` regardless of parent shell.
- RAGAS 0.4.3 `from ragas.metrics import context_precision` DeprecationWarning (API evolution).
- `comparison.md` emits `**Embedder by tier:**` (markdown bold) instead of `## Embedder by tier` (heading). No user-visible impact; verifier regex was wrong, compare.py was RAW-LOCKED.
- `compare.py` cost-window selection: prefer ts-window grep over `ls -t | head -1` to avoid pytest test-bleed pollution.
- v1.1 deferred requirements (carried over from `REQUIREMENTS.md` v1.1 section): CLEANUP-01 (consolidate `tier-N-name/` ↔ `tier_N_name/` shim folders), CLEANUP-02 (`[debug-tier5]` extras), METH-01 (bootstrap 95% CI), METH-02 (full cross-judge re-scoring), METH-03 (per-tier per-class min/max/stdev), BLOG-01 (rebuttal run pipeline).
