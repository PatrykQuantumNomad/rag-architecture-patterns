    **Core loop:**

    4. `def parse_missing_papers(papers_dir: Path, output_root: Path, manifest_path: Path, paper_ids: tuple[str, ...] = MISSING_PAPERS_GOLDEN, console: Console | None = None) -> tuple[int, list[str]]`:
       Returns `(n_parsed, failed_paper_ids)`.
       - by_id = `_load_papers_manifest(manifest_path)`
       - for pid in sorted(paper_ids):
         - if pid not in by_id: failed.append(pid); console.print red "Paper {pid} not in manifest"; continue
         - target_dir = output_root / pid
         - if target_dir.exists(): console.print dim "Skipping {pid} (already parsed at {target_dir})"; continue
         - pdf = papers_dir / by_id[pid]["filename"]
         - if not pdf.exists(): failed.append(pid); console.print red "PDF missing at {pdf}"; continue
         - cmd = `_build_mineru_command(pdf, output_root, pid)`
         - console.print dim f"$ {' '.join(cmd)}"
         - try: subprocess.run(cmd, check=True, capture_output=False); n_parsed += 1
         - except subprocess.CalledProcessError as exc: failed.append(pid); console.print red f"MineRU failed for {pid}: {exc}"; continue
       - return (n_parsed, failed)

## Trust Boundaries

| Boundary | Description |

| Sandbox env → MineRU subprocess | The sandbox actively blocks MineRU at the OS level (Pitfall 3). Detection is a soft-fence; the OS-level block is the hard fence. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-02-01 | T (Tampering) | Command injection via crafted `filename` field in `papers.json` (e.g., `"x.pdf; rm -rf /"`) | mitigate | `subprocess.run` is invoked with list-form args (no `shell=True`); the malicious filename would be passed as a single argument to `mineru -p`, which would fail to open it. V5 ASVS list-form invocation. The papers manifest is curated (committed to git, reviewed at checkin); not user-uploaded at runtime. |
| T-02-02-02 | I (Info disclosure) | MineRU subprocess stderr leaks PDF content via debug logs | accept | MineRU's stderr is verbose but contains no secrets. The subprocess inherits the user's environment (including OPENROUTER_API_KEY) but does NOT call any LLM API — `mineru` is pure local work. The key is irrelevant to this subprocess. |
| T-02-02-03 | D (DoS) | Adversarial PDF with malformed table/image causes MineRU to OOM or hang | accept | Mitigated by per-paper isolation: a hang on paper N does NOT poison the loop's other papers. User can kill the script with Ctrl-C; script logs the partial state; idempotent re-runs skip already-parsed. |
| T-02-02-04 | T (Tampering) | User runs script in sandbox despite the pre-flight; MineRU half-runs and corrupts a per-paper output directory | mitigate | Pre-flight refuses by default (return 2 before any subprocess call); `--allow-sandbox` is a deliberate override, not an accident. | T-02-02-05 | E (Elevation of privilege) | `--allow-sandbox` lets a malicious caller bypass the safety check | accept | The pre-flight is a UX safety net, not a security boundary. The actual security boundary is the kernel-level OMP shmem block, which `--allow-sandbox` cannot bypass. A malicious caller already has shell access; the flag is not a privilege boundary. |
| T-02-02-06 | I (Info disclosure) | Mineru downloads model weights from HuggingFace; downloaded files could include trojaned models | transfer | Trust transferred to HuggingFace + the upstream `mineru` PyPI package. The repo's `pyproject.toml` pins `mineru>=2.5,<4` (resolved 3.1.4) which is a curated open-source release; supply-chain risk is at the same level as any other Python dep.
